PROJECT 01

난독화된 한글 리뷰 복원 AI

AI 자연어처리(NLP) 엔지니어 집중 과정 3기 | 2026.01

담당모델 비교 실험·선정 · JAMO 전처리 설계 · Noise Type Classifier 구현 · 태스크 분기형 아키텍처 설계

야민정음과 음운 오류는 복원 방향이 근본적으로 다릅니다. 단일 모델로 두 유형을 처리하면 규칙이 충돌하고 출력이 불안정해집니다.

“문제는 모델 성능이 아니라 태스크 정의였습니다.”

야민정음은 자모 단위 모양 치환이고, 음운 오류는 발음 기반 표기입니다. 두 오류를 단일 모델에 입력하면 어느 규칙을 따를지 결정하지 못해 출력이 불안정해집니다. 더 큰 모델보다 판단 구조를 외부에서 만드는 방향을 선택했습니다. 오류 유형을 먼저 분류하고 전용 복원 경로로 분기하는 구조가 핵심이었습니다.

DECISION TRAIL

시도LLaMA 3.1-8B — decoder-only 구조라 복원보다 요약·재작성 경향이 강해 원문 재현에 부적합했습니다.
시도KoBART 단독 — BPE tokenizer가 야민정음 자모를 올바르게 분해하지 못해 입력 자체가 손상되었습니다.
해결JAMO 전처리 도입 — 초성·중성·종성으로 분리 후 KoBART에 입력, 토큰 손상 해소 + 과생성 억제 효과 확인.
한계CharCNN 보조 신호 — 야민+음운 혼합 입력에서 어느 복원 규칙을 따를지 모델이 판단하지 못해 출력 일관성 무너짐.
한계Special Tag 방식 — KoBART가 태그를 조건 신호가 아닌 일반 토큰으로 처리해 태그 자체가 변형·소실됨. 입력 주입 방식 자체가 부적합.
최종KoELECTRA Noise Classifier로 유형을 먼저 분류 → 각 유형 전용 KoBART로 분기. Custom Dictionary + JAMO 조합으로 야민·그래픽 복원 안정화.

lovelop — 상권분석 AI 시뮬레이션 SaaS

AI 자연어처리(NLP) 엔지니어 집중 과정 3기 | 2026.02

담당데이터 분석·전처리 · AI Agent 페르소나 설계 · 시뮬레이션 환경 설계·구축·고도화·검증 · PPT 제작

기존 상권분석 서비스는 과거 매출 데이터를 보여줍니다. 자영업자가 실제로 필요한 것은 "이 변화를 적용하면 어떻게 될까"입니다.

“두 질문은 다릅니다. 후자에 답하려면 예측이 아니라 시뮬레이션이 필요합니다.”

lovelop(local + develop)은 식당·카페 자영업자를 위한 웹 기반 AI 상권분석 SaaS입니다. ① lop진단: 공공·민간 정형 데이터(매출/상권/인구)와 비정형 데이터(리뷰)를 수집해 GPT-5.2 2-Stage 구조로 강점·약점 분석 및 솔루션 9가지 제안. ② 시뮬레이션: 망원동 상권 데이터 기반 160개 AI 페르소나(방문 목적 4유형 × 세대 5종 × 성별·인원 구성)가 Before/After 두 번 실행. ③ lop검증: 11가지 지표로 전후 비교 리포트 생성.

Agent 5-Step 의사결정: ① 망원동 외식 여부 결정 → ② 업종 선택 → ③ 매장 선택(평점·키워드·타 에이전트 리뷰 참조) → ④ 평가·리뷰 생성(세대별 말투) → ⑤ 식후 행동 결정(카페 방문 시 ③ 재진입). 페르소나 160개는 단순 인구통계 조합이 아닌 방문 동기·가격 민감도·재방문 기준 등 행동 파라미터를 세대별로 다르게 설정했습니다.

PROJECT 03🏆 장려상 · 팀 리더

KillKong — 콩글리쉬 교정 AI Agent

포스코 AI·BigData 아카데미 30기 | 팀 A4조 (5인) | 2025.07 – 2025.08

담당팀 리더 · 전체 파이프라인 설계 · 모델 선정 실험 · LoRA 파인튜닝 · RAG 시스템 설계 · 개인화 메모리 아키텍처

GPT-5는 "black consumer"를 인종 표현으로 해석했습니다. "after 신청했어?"의 한국식 의미도 처리하지 못했습니다.

“범용 LLM의 한계는 성능이 아니라 도메인 커버리지. 한국인 특화 오류 패턴은 전용 DB 없이는 해결되지 않습니다.”

3단계 고도화로 시스템을 구축했습니다. Stage 1 Qwen 단독(톤은 자연스럽지만 콩글리쉬 미인식) → Stage 2 +RAG(skinship→physical, selca→selfie 교정) → Stage 3 +Memory(반복 오류 일관 교정·오답노트). 모바일 배포가 목표였기 때문에 경량화는 선택지가 아니었습니다. LoRA + 4-bit 양자화로 14GB를 3.5GB로 줄이면서 BertScore 0.95를 유지했습니다.

LoRA + 4-bit 양자화로 14GB → 3.5GB(76%↓) 경량화. 성능 손실 우려와 달리 BertScore 0.95 유지. 모델 크기보다 도메인 특화 데이터 품질이 성능에 더 직접적으로 영향.

DECISION TRAIL

시도T5 / Flan-T5 — Friends 대사 59,587문장으로 파인튜닝. RAG retrieval 결과를 강제 반영하지 않아 두 표현 교정 중 하나만 처리됨.
문제초기 마스킹 방식 실패 — [MASK]가 있을 때만 단어를 생성하는 경향 확인. 태스크 정의 자체를 재설계.
전환Qwen-2.5(1.5B) + system/user/assistant 대화형 포맷 — 지시문 기반 교정 준수율·콩글리쉬 인식 정확도·응답 스타일 제어 모두 Flan-T5 대비 우수.
전략데이터 품질 우선 — 초기 2,000개 콩글리쉬 DB를 맥락·권장 치환어 포함 630개로 정제. 수량보다 품질이 성능에 직접적으로 영향.
최종Stage3: Qwen + RAG(FAISS+BM25) + 개인화 Memory — 반복 오류 일관 교정 및 오답노트 기능 구현.

PROJECT 04

FinView — GPT-4 기반 자동 재무 분석 리포트

성신여자대학교 | 2024.09 – 2024.11 | 팀장

담당전체 파이프라인 설계(팀장) · DART+뉴스 데이터 수집 · 재무 비율 계산 · XGBoost + KMeans · GPT-4 프롬프트 엔지니어링 · 환각 검증 로직

LLM에 재무 수치를 직접 입력하면 계산 오류가 발생합니다. GPT-4는 숫자를 처리하는 모델이 아니기 때문입니다.

“분업이 필요했습니다. 수치 계산은 구조화 모델이, 언어 생성은 LLM이 담당하는 파이프라인을 설계했습니다.”

7단계 파이프라인: ① DART + 뉴스 수집 → ② 12개 재무 비율 계산 → ③ XGBoost A~D 등급 분류(91% 정확도) → ④ KMeans 유사 재무 구조 기업 군집화 → ⑤ 뉴스 리스크 태깅(TF-IDF + SMOTE) → ⑥ GPT-4 리포트 생성(군집 평균 패턴 주입) → ⑦ D3.js 대시보드 시각화. XGBoost가 수치를 계산하고 GPT-4가 언어로 해석하는 역할 분리가 수치 환각을 80% 이상 개선한 핵심입니다.

PROJECT 05

CT→MRI 의료영상 교차-모달리티 변환

딥러닝연구개발소 (한성대학교 단기근무 연구원) | 2023.06 – 2023.09

담당도메인 변환 알고리즘 전체 설계 · 모델 선정 비교 · Generator·Discriminator 구조 튜닝 · Loss 함수 조합 설계 · 의료진 피드백 반영

CT와 MRI는 같은 시점에 같은 환자를 찍은 paired 데이터가 병원 환경에서 존재하지 않습니다.

“paired 학습 기반 모델은 적용 불가능. unpaired 데이터 800장이 주어진 전체 조건이었습니다.”

ResNet 블록 수는 6개(디테일 손실)·12개(메모리 초과)·9개(최적) 순으로 실험. 초기 학습에서 Discriminator가 빠르게 수렴해 mode collapse 발생 → 학습률을 G:0.0002 / D:0.0001로 분리, 업데이트 비율 G 2회:D 1회로 안정화. Loss 가중치 Adversarial:Cycle:Perceptual:Identity = 1:10:2.3:0.5는 Cycle 비중을 높여 해부학적 구조 유지를 우선한 결과. 의료진 피드백("뼈 경계 흐림")을 반영해 Loss 비율을 반복 재조정했습니다.

DECISION TRAIL

불가Pix2Pix — paired 데이터 필수. 실제 병원 환경에서 CT·MRI 1:1 매칭 불가.
부적합StyleGAN — 스타일 변환 특화. 뼈·장기 같은 해부학적 구조 보존이 어렵다.
부족VAE — 재구성 품질이 의료 영상 수준에 미치지 못함.
채택CycleGAN — unpaired 학습 가능 + Cycle Consistency Loss로 구조 보존 강제. 유일하게 제약 조건을 모두 충족.

ARCHITECTURE

Generator 아키텍처 (ResNet-9)
Discriminator 아키텍처 (PatchGAN)
